# Devlog: 2026-04-24 — Post-Playtest Fix Plan

## Context

First playtest of Gate 1 prototype. Card color system validated (player can read card modes). But 3C feels bad across the board, enemies are visually indistinct, player dies too fast, and card variety is too low.

This document provides specific fixes with exact values for the tech session.

---

## P0 — Fix 3C (highest priority)

### Problem: "Everything feels slightly off"

After reviewing player.gd (394 lines), identified specific issues:

### Fix 0a — Camera effect architecture check

On line 288, camera effects SET camera.rotation.x directly. Head stores mouse pitch, Camera (child of head) stores effects as local rotation. This SHOULD work if all offsets recover cleanly to 0. If any offset lingers, the camera drifts — explaining "mouse look feels off."

**Action:** Debug log all camera offsets at rest. If any non-zero when idle, fix recovery paths. Ensure move_toward snaps to exactly 0.0 below threshold (0.001 rad). Avoid lerp for recovery (never reaches 0).

### Fix 0b — Movement feels floaty AND sluggish

"Sluggish" = slow to START. "Floaty" = slow to STOP. Ground values (50 accel/decel) should be fine. Real culprit is AIR control:
- air_decel = 5 means 1 SECOND to stop in air
- air_control_factor = 0.3 means airborne speed cap is 1.5 m/s
- Any geometry bump makes player lose responsiveness

**New values:**

    ground_accel = 80.0     # was 50
    ground_decel = 80.0     # was 50
    air_accel = 30.0        # was 15
    air_decel = 20.0        # was 5
    air_control_factor = 0.6 # was 0.3

### Fix 0c — Jump feels wrong

Current: jump_velocity=5.0, gravity=20.0. Actual height = 25/40 = 0.625m (knee-height). 3C doc said 1.25m but math was wrong. Need v = sqrt(2*20*1.25) = 7.07.

**Fix:** jump_velocity = 7.0 (was 5.0). Keep gravity at 20 for snappy landings.

### Fix 0d — Firing feels weak

No audio, no weapon model, 1.5 degree kick only. Feels like a laser pointer.

**Fixes in priority order:**
1. Add placeholder fire audio (heavy revolver crack). #1 impact on feel.
2. Build basic weapon model from CSG (box body + cylinder barrel in lower-right).
3. Increase recoil: recoil_kick_degrees = 3.0 (was 1.5), recovery = 0.15s (was 0.2).
4. Weapon viewmodel kick: translate Z +0.05 over 0.05s, return over 0.1s.
5. Hit confirmation audio: distinct thwack on enemy hit.
6. Try small screen shake: 0.5-1 degree random, 0.05s. Revert if hurts card readability.

### Fix 0e — ADS transition clunky

**Fix:** ads_transition_speed = 10.0 (was 6.67 = 0.1s instead of 0.15s). ADS should shift weapon model forward/centered if model exists.

### Fix 0f — Spatial awareness poor

Arena is all gray. No differentiation.

**Material colors:**
- Floor: (0.15, 0.15, 0.18) dark
- Walls: (0.25, 0.25, 0.28) medium
- Cover: (0.3, 0.25, 0.2) brownish
- Pillars: (0.2, 0.2, 0.25) blueish gray
- Platform edges: (0.4, 0.3, 0.2) warm accent
- Add emissive strips on cover tops

## P1 — Enemy Visual Distinction

### Problem: Player only noticed 1 enemy type

All 3 use similar capsule meshes. Indistinguishable at 10-30m.

### Fix: Different silhouettes

| Enemy | Current | Fix |
|---|---|---|
| Grunt | Red capsule 1.8m | Add BoxMesh "gun" extending forward. Dark red. |
| Big Eye | Purple sphere 1.0m | Scale to 1.5m radius. Brighter purple. Pulse eye light. Float 0.5m. |
| Rusher | Orange capsule 1.5m | Wider (0.5m radius, 1.2m height). Bright orange. Lean 15 deg. Speed 9 (was 7). |

### Enemy health bars

- 1m wide billboard bar above each enemy
- Dark gray background, red fill (HP%)
- 0.3m above top
- Only visible after taking damage

### Status effect visibility

- Poison label: 2x font size, bold green, black outline
- Poison glow: range 3->5, increase energy
- Burn particles: bigger, brighter, more
- Add colored ground ring (green=poison, orange=burn)
