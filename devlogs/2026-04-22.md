# Devlog: 2026-04-22 — Project Discovery & Design Direction

## Context

Analyzed the predecessor project **PES** (`~/PES`) — a Godot 4.6 extraction shooter (~7,850 lines GDScript, 14 design docs). Goal: understand everything before rebuilding as a new project (GP1).

---

## PES Analysis Summary

### What PES was
- 3D first-person extraction shooter, single-player
- Inspired by HOLE (NEGAFISH), Marathon, The Division 2, Tarkov
- PSX shader aesthetic (liminal horror / backrooms vibe)
- Core loop: spawn -> enter microwave portal -> fight enemies -> loot -> extract or die

### What PES had (code inventory)

| System | Lines | Quality |
|---|---|---|
| Player controller | 1,222 | God-class, 8 concerns, 6 duplicated match blocks, but movement feels good |
| Enemy FSM (8-state) | 814 | Monolithic but cover scoring is well-tuned (9 weighted factors) |
| Squad Manager (Encounter Director) | 276 | Clean Division-2-style: SETUP/ENGAGE/PUSH/FALLBACK phases, engagement slots, fireteams |
| Cover Builder (@tool) | 268 | Edit-time baking, good pattern |
| Inventory (5x4 grid) | 256 | Clean autoload, Tarkov-style |
| Inventory UI | 482 | Built in code (should be .tscn), dead paths |
| Item Database | 182 | Hardcoded registry, positional-arg factories |
| Loot system | 257 | Spawner + item + drop tables |
| Stash Manager | 85 | JSON persistence, no atomic write |
| PSX Shaders | 2 files | Surface + post-process, well-tuned |
| Microwave Portal | 127 | Area3D + hold-E + state machine |
| Tactical Minimap | 140 | F3 debug overlay |
| Spawn Closet | 120 | LD-placed enemy spawner |
| Other (VFX, builder, etc.) | ~600 | Support scripts |

### Architectural problems found
1. God-classes (player 1,222 lines, enemy 814 lines)
2. Magic numbers everywhere (air control 0.3, headshot Y 1.3, etc.)
3. String-based `root.set("prop", val)` — no type safety
4. Duplicate constants across files (GRID_W/H in inventory + UI)
5. Private member access across files (tactical_map reads enemy._cover_point)
6. Hardcoded absolute node paths ("/root/WalkScene/WalkthroughUI")
7. Shared tween channels that conflict (jump/kick/reload/land all fight)
8. No data-driven content (all items, weapons, archetypes hardcoded)
9. No tests

### Dead/broken features in PES
- **Jam system**: UI, input binding, and ItemData field exist — zero gameplay wiring
- **Consumable heal**: `_use_consumable` decrements quantity, never applies heal
- **Drop items**: `_drop_item` removes from inventory, never spawns world loot
- **`inventory_full` signal**: emitted, zero listeners — silent failed pickups
- **`extraction_started` signal**: fires every frame (bug)
- **Stash `SAVE_VERSION`**: written but never read — no migration path

### What was good in PES (worth preserving conceptually)
- Encounter Director pattern (engagement slots, fireteams, phases)
- 3-phase peek system (PEEK_OUT / PEEK_SHOOT / PEEK_RETURN)
- Cover scoring formula (9 weighted factors, well-tuned)
- Coyote + buffer jump (textbook clean)
- PSX shader pipeline (surface + post-process)
- CoverBuilder @tool editor pattern
- Design documentation culture (14 docs, ~4,000 lines of thinking)

## Design Direction Exploration

### Directions considered and rejected

**Direction 1 — Horror extraction (committed horror)**
- Rejected because: smooth 3C movement (which feels good) directly fights horror. Horror needs vulnerability; smooth movement creates power.

**Direction 2 — Arcade extraction roguelite**
- Rejected because: no unique hook. "Extraction shooter" is a saturated market (Tarkov, Hunt, Marathon dominate).

**Direction 3 — HOLE-like weirdcore**
- Rejected because: drops the AI investment (~2,500 lines) and the shooter loop entirely.

**Direction 4 — Backrooms extraction**
- Rejected after discovery: found "Backrooms: Extractions" on Steam — early access, mixed reviews, 10-100 concurrent players. Backrooms trend peaked ~2022. The niche is small and cooling.

### The hook problem
Core issue identified: PES had no unique hook. It was a mashup of references ("Tarkov meets HOLE meets Division with PSX graphics") rather than an original idea.

Commercial requirement: a 5-second trailer hook that strangers understand without explanation.

### Direction chosen: FPS Roguelite Deckbuilder

Inspired by weeks of playing Slay the Spire. Key insight: the "build" (constructing a powerful deck through choices) is the hook, not the combat itself.

**Initial concept:** merge deckbuilding with FPS — strategic card choices between rooms that modify combat.

**Evolved concept (the real hook):**

> "Your deck is your magazine. Each shot fires a card."

This is structurally different from existing FPS roguelites (Gunfire Reborn, Roboquest) where cards/scrolls are passive modifiers. In this game, cards are ACTIVE, CONSUMABLE, SEQUENTIAL. Every trigger pull is a micro-strategy moment. Reloading becomes reshuffling.

## "Cards Are Bullets" — Core Concept

### The pitch (5-second version)
Your deck is your magazine. Each shot fires a card. Reload to reshuffle.

### What makes it unique
- Every trigger pull plays a card effect (not a passive modifier)
- Reloading = reshuffling the deck (transforms worst FPS moment into strategic beat)
- Deck construction between fights determines combat options
- Strategy lives in the deck; reflexes live in the shooting

### How existing PES code maps to this

| PES System | Roguelite Reuse |
|---|---|
| Player controller (smooth 3C) | Foundation — movement IS the game |
| Squad AI (Director + FSM) | Enemy palette for encounter design |
| Engagement Director phases | Encounter scripts / room difficulty scaling |
| SpawnCloset prefabs | Room builder — each room = configured closets |
| Inventory 5x4 grid | Card/loadout display UI |
| Item Database | Card database |
| Stash persistence | Meta-progression unlocks |
| PSX shaders | Visual identity (stays) |
| Microwave portal | Floor exit / extract with current rewards |
| Cover builder | Level geometry tool for hand-designed rooms |
| 5 weapons with ranges | 5 weapon archetypes that interact with cards |

### Card design: 4 archetypes, 20 starter cards

**BURN (fire DoT build):**
1. Ignite Rounds — 20% chance to ignite, 2 dmg/sec for 3s
2. Flamethrower Mod — guaranteed ignite, range halved
3. Wildfire — burning enemies spread fire within 3m
4. Cremate — burning enemies explode on death (50 dmg, 4m)
5. Phoenix — kill burning enemy = +1 HP + fire resistance

**PRECISION (headshot/crit build):**
6. Hunter's Eye — headshots +50% bonus
7. Steady Aim — ADS spread = 0, ADS move -50%
8. Execute — headshots on <30% HP = instant kill
9. Chained Kill — kills refund 1 bullet + reset reload
10. Sniper's Patience — ADS hold charges damage up to +200%

**SHRAPNEL (pellet/chaos build):**
11. Buck Magnum — all weapons fire as shotguns (reduced per-pellet dmg)
12. Pellet Trickery — missed pellets 30% chance to return and re-fire
13. Cluster — same-target pellets +20% per pellet (max +200%)
14. Ricochet — pellets bounce off walls 3 times
15. Trigger Discipline — fire rate -70%, damage +400%

**MOMENTUM (movement/air build):**
16. Airborne — airborne damage +50%
17. Dashing — gain dash (2s cd), invincible during, instant reload
18. Hot Streak — consecutive kills within 3s = +10% dmg (stacks 5x)
19. Bullet Time — kills slow time for 0.5s
20. Glass Cannon — HP capped at 50, damage +150%, dash cd -50%

Cross-archetype interactions exist (e.g., Ignite Rounds + Buck Magnum = every pellet rolls ignite).

**Note: these 20 cards were designed as passive modifiers BEFORE the "cards are bullets" idea.** They need to be redesigned as per-shot effects for the new framework. The archetypes and synergy webs are still valid; the implementation changes.

## Open Design Questions

These need answers before prototyping. Listed by priority.

### Q1 — Magazine size
- 6 cards (revolver, each shot precious) vs 15 (AR, combo potential) vs 30 (LMG, full deck play)
- Variable per weapon? Central pacing knob.

### Q2 — What does "a card" do when fired?
- Replace projectile behavior? (poison bullet, ricochet)
- Modify weapon temporarily? (next 3 shots pierce)
- Trigger an event? (summon turret, heal)
- All of the above?

### Q3 — Shuffle model (CRITICAL)
- **Path A: Pure random** — chaotic, no strategic construction
- **Path B: Shuffled deck (StS model)** — know contents not order, draw through full deck before reshuffle
- **Path C: Hand-drafted** — player arranges order pre-fight
- **Path D: Hybrid** — StS-style draw/discard with cards that influence deck order
- Recommendation: Path B or D. Pure random (A) kills ownership. Hand-drafted (C) breaks FPS pace.

### Q4 — How do cards enter the deck?
- Pick from 3 after fights (StS model)?
- Loot drops from enemies?
- Shop purchase?
- All?

### Q5 — Do cards stack or stay unique?
- Multiple copies = tune deck density (70% power, 30% filler)
- Unique only = each card precious
- StS uses multiple copies. Probably should too.

### Q6 — Basic/filler cards
- "Weak bullets" that come free — like StS Strikes/Defends
- Can players thin basics? (StS deck thinning is a KEY mechanic)
- Do basics do anything useful or are they dead weight?
- If 80% of shots are basics, 80% of combat feels mundane. This is a real problem.

### Q7 — Deck visibility during combat
- Full magazine visible (strategic but busy)
- Next 1-2 cards only (minimal, some foresight)
- Nothing (pure surprise, reactive)
- Optional toggle (hold TAB to inspect)
- UX problem: FPS players want clean screen; deckbuilders need information

### Q8 — Weapons and cards interaction
- Universal deck (all weapons fire same cards)
- Per-weapon decks (separate)
- Cards have weapon-type restrictions
- Weapons ARE cards (radical: pick up M77 card, fire it once as a mega-shot, consumed)

## Identified Risks

1. **"Every shot feels the same" trap** — if 80% of cards are basics, 80% of shots are mundane
2. **Predictability vs chaos** — FPS players expect consistent guns; this game deliberately breaks that
3. **Card-enemy balancing** — can't design "boss needs armor-piercing" if player might not draw it
4. **UI information overload** — current FPS UI is crosshair + ammo; card system needs 5x more info
5. **Content volume** — consumable cards need 200-300 designs for full game (StS has 350+)
6. **Balatro shadow** — card roguelite will be compared to Balatro; need to own the FPS side strongly
7. **Bad RNG frustration** — garbage draws before boss = feels unfair (need mitigation: deck thinning, reserve system)

## Competitive Landscape

### FPS Roguelites (validated market)
- **Gunfire Reborn** (4M+ copies) — scrolls as random passive modifiers, co-op, cel-shaded
- **Roboquest** (Very Positive, ~50k reviews) — between-room card picks, stat bumps
- **ULTRAKILL** (~1M copies) — weapon variants, style meter, solo dev success story
- **Neon White** — cards ARE weapons (discard to double jump), speedrun-focused
- **BPM: Bullets Per Minute** — rhythm FPS roguelite, cult hit

### Market gap identified
"Strategic FPS roguelite with real deck construction" is underserved. Gunfire Reborn cards are random drops. Roboquest cards are mostly stat bumps. Nobody has done:
- Per-shot card effects (not passive modifiers)
- StS-style deliberate deck construction
- Dark/atmospheric aesthetic (most competitors are bright)

### Positioning
> "A slower, darker FPS roguelite where you BUILD your deck through deliberate choices — not random drops. And your deck IS your ammunition."
